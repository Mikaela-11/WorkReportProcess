# WorkReportProcessor  
業務用 日報・週報 自動集計システム

---

## 概要

WorkReportProcessor は、  
業務現場で発生する **不完全・不統一な作業記録データ** を前提に、  
日報・週報を自動生成することを目的とした  
業務処理型アプリケーションです。

本プロジェクトでは、  
**安定性・保守性・役割分離** を重視した設計を意識しています。

---

## 想定利用シーン

- 手作業で日報・週報を作成している業務
- 入力フォーマットが統一されていない現場
- 将来的な自動化・AI連携を見据えた基盤構築

---

## システム構成

```text
Input
 ↓
Validation（入力検証・補正）
 ↓
Logic（業務集計処理）
 ↓
Output（人が読める形式で出力）
```

---

## ディレクトリ構成

```text
src/
 ├ validation/
 │   ├ validator.py
 │   └ result.py
 ├ logic/
 │   ├ aggregator.py
 │   ├ daily_report.py
 │   └ weekly_report.py
 ├ output/
 │   ├ daily_writer.py
 │   └ weekly_writer.py
 └ main.py
 ```

⸻

特徴①：Validation 層（入力検証）

本システムでは、
「入力が常に正しいこと」を前提としません。
	•	フィールド欠損
	•	型不正
	•	業務的に不正な値

これらを Validation 層で検知し、
処理を止めずにスキップまたは補正する設計としています。

⸻

特徴②：Logic 層（業務処理）

Validation 済みのデータのみを対象に、
	•	日別集計（日報）
	•	ユーザー別・タスク別集計（週報）

を行います。

入力処理・出力処理と分離することで、
業務ロジックの変更に強い構造を実現しています。

⸻

特徴③：Output 層（可読性重視）

出力結果は、
そのまま業務で確認できる形式 を想定しています。
	•	日報：日付単位の作業内容と総工数
	•	週報：担当者ごとの作業内訳と合計時間
